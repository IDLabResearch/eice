Pathfinding Service

Discover paths between any two DBPedia resources

About the Pathfinding Service

Graph algorithms make optimal use of available computation resources to and paths in structured data. Linked Data allows web services to make these approaches generic. However, scalability has always been a major issue in this context. To overcome this issue the Pathfinding Service makes use of an approach by pre-processing datasets and indexing them. It can find paths between Linked Data resources in DBpedia in most cases within a couple of seconds (in future releases other large datasets will be supported as well).

Use cases

Everything is Connected

Everything is Connected is able to find a path between a Facebook user and any concept known on DBpedia, such as, persons, locations, things, etc. The found path is presented to the end-user as a short story, which explains the relation between the searched concept and the end-user.

This service was designed and implemented by MultiMediaLab | Ghent University | iMinds

Service API

build v20121129.1

/paths?from=[source_URI]&to=[destination_URI]

Returns path in JSON between [source_URI] and [destination_URI]

/subjects?label=[comma seperated list of keywords](&type=[type of keywords])

Returns list of URI's with their connectivity to other resources based on inputted list of keywords and optional type. Including the type improves the accuracy of the returned URIs.

/description?uri=[comma separated list of URIs]

Returns list in JSON with description of the URIs (labels and abstracts).

/visualization

Returns a HTML page with an interactive visualization of recent found paths.

/metrics

Returns a HTML page with the latest statistcs on recent pathfinding queries such as hitrate and execution times.
